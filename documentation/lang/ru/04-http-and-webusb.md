# 4. HTTP и WebUSB

## 4.1 HTTP (Ethernet, LwIP)

**Реализация:** `Src/task_net.c`. **Порт:** 80. **TLS:** нет.

### Кратко

| Параметр | Значение |
|----------|----------|
| База | `http://<ip-устройства>/` |
| Макс. запрос | Должен помещаться в буфер приёма (порядка 1 KiB; см. `REQ_BUF_SIZE` в коде) |
| Таймаут приёма | Порядка 2 с (см. код) |

### Маршруты

| Метод | Путь | Ответ |
|-------|------|--------|
| GET | `/ping` | `200` `text/plain`: `pong` |
| GET | `/` | `200` HTML-демо |
| GET | `/tx/signed` | `200` JSON статус / подпись |
| POST | `/tx` | `200` HTML (см. оговорку ниже) |
| * | прочее | 404 или заглушка |

### `POST /tx`

- **JSON:** в `Content-Type` должно быть `application/json`. Поля тела: `recipient`, `amount`, `currency` (наивный парсер).
- **Форма:** `application/x-www-form-urlencoded`.

Валидация через `tx_request_validate()`. **Оговорка:** HTTP-слой может вернуть успешный HTML даже если постановка в очередь не удалась — **всегда** используйте `/tx/signed` или UI устройства как источник истины.

### `GET /tx/signed`

Примеры JSON:

```json
{"status":"signed","sig":"<hex>"}
```

```json
{"status":"pending"}
```

## 4.2 WebUSB

**Файлы:** `Core/Src/usb_device.c`, `usb_webusb.c`, `usbd_conf_cw.c`, `usbd_desc_cw.c`.

- Протокол для операций кошелька, дополняющий HTTP.
- В дескрипторах есть WebUSB platform UUID (см. исходники).
- Та же модель доверия: **нет** аутентификации транспорта; подтверждение на устройстве.

## 4.3 Правило для оператора

Считайте **Ethernet + HTTP** только **доверенной LAN**. Нет конфиденциальности и аутентификации запросов по сети — физический доступ и размещение в сети входят в вашу модель угроз.
