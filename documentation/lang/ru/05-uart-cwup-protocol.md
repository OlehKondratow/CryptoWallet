# 5. UART — CWUP-0.1 (протокол CryptoWallet UART)

**Реализация:** `Core/Src/cwup_uart.c`. **Типичная продуктовая сборка:** `USE_WEBUSB=1`, `USE_RNG_DUMP=0` — CWUP на USART3 вместе с WebUSB на PA11/PA12.

**Область:** лабораторная диагностика, CI и (по спецификации) фреймовый TRNG. **Не** обновление прошивки по UART (это загрузчик). **Не** замена HTTP/WebUSB как основного канала пользователя.

## 5.1 Контекст цепочки загрузки

```
ROM → (опционально stm32_secure_boot) → приложение CryptoWallet
```

CWUP существует **только после** запуска FreeRTOS. Трафик во время работы загрузчика — **не** CWUP.

## 5.2 Фазы строк

| Фаза | Смысл | Вид с хоста |
|------|-------|-------------|
| A | До готовности CWUP | Может быть текст; CWUP ещё не готов |
| B | `CW+READY,proto=CWUP/0.1,...` | CWUP объявлен |
| C | Текстовый режим: строки `AT+` / `CW+` | Команды по §5.4 |
| D | Бинарный фреймовый TRNG (§5.5) | После `AT+RNG=START` — в прошивке **не реализовано** |

`rng=` в `CW+READY` относится к **фреймовому** TRNG — **не** к сырому потоку `USE_RNG_DUMP`.

## 5.3 Режимы

| Режим | TX | RX |
|-------|----|----|
| TEXT | Строки `\r\n` | `AT+...` |
| BINARY_RNG | Только фреймы §5.5 | `AT+RNG=STOP` / таймаут |

Правило: в BINARY_RNG не выводить `printf`-логи на тот же UART.

## 5.4 Команды (MVP)

Формат: `AT+<ГЛАГОЛ>[=<аргументы>]\r\n` → `CW+<КОД>[,полезная нагрузка]\r\n` или `CW+ERR=<n>,<msg>\r\n`.

| Команда | Назначение |
|---------|------------|
| `AT+CWINFO?` | Протокол + информация о сборке |
| `AT+FWINFO?` | Строка целостности приложения (как `fw_integrity_snprint()` / лог `FWINFO`) |
| `AT+BOOTCHAIN?` | Человекочитаемая цепочка загрузки (лаборатория) |
| `AT+MARKS` | Маркеры CI — **не реализовано** |
| `AT+RNG=START,<bytes>` / `AT+RNG=STOP` | Сессия фреймового TRNG — **не реализовано** |
| `AT+PING` | `CW+PONG` |
| `AT+READY?` | Повтор `CW+READY` |
| `AT+WALLET?` | Лаборатория: флаги `seed` / `crypto_sign` — **без секретов** |
| `AT+SELFTEST?` | Тик / проверка |
| `AT+ECHO=<text>` | Печатный ASCII — регрессия CI |

Запись в кошелёк остаётся на HTTP/WebUSB — не CWUP-0.1.

## 5.5 Формат фрейма TRNG (спецификация; парсер на хосте)

```
[0..1]   magic 0xC7 0x57
[2..3]   u16 payload_len N (LE, N ≤ 256)
[4..3+N] N байт TRNG
[4+N..7+N] u32 CRC32 (IEEE) по [0 .. 4+N-1]
```

## 5.6 Статус реализации (прошивка)

| Элемент | Статус |
|---------|--------|
| `CW+READY`, `AT+PING`, `AT+CWINFO?`, `AT+FWINFO?`, `AT+BOOTCHAIN?`, `AT+WALLET?`, `AT+SELFTEST?`, `AT+ECHO=` | Реализовано |
| `AT+MARKS`, фреймовый TRNG §5.5 + `AT+RNG=*` | **Не реализовано** |
| `USE_RNG_DUMP=1` | Сырой бинарный UART — **Cwup_Init** не запускает CWUP |
