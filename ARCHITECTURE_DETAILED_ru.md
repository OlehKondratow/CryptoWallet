# Архитектура и сравнение стека технологий

**Визуальное руководство: stm32_secure_boot vs CryptoWallet**

---

## 🏛️ Сравнение архитектуры системы

### Архитектура stm32_secure_boot

```
┌─────────────────────────────────────────────────────────────────┐
│                     АППАРАТНЫЙ СЛОЙ                             │
│  STM32H743 (Cortex-M7 @ 480 MHz, 2MB Flash, 1MB SRAM)         │
│  Периферия: UART, USB, I2C, GPIO, RNG (опцион.), Ethernet    │
└─────────────────────────────────────────────────────────────────┘

                    ↓ Последовательность загрузки ↓

┌─────────────────────────────────────────────────────────────────┐
│          ЗАГРУЗЧИК (64 KB @ 0x08000000)                          │
│                                                                  │
│  ┌─ Код запуска (сборка ARM Cortex-M7)                        │
│  ├─ Конфигурация тактирования (PLL, делители AHB)            │
│  ├─ Инициализация HAL                                          │
│  ├─ Вычисление SHA-256 (Хеш образа)                           │
│  │   └─ hash = SHA256(app_image)                              │
│  ├─ Проверка подписи ECDSA                                    │
│  │   ├─ С библиотекой CMOX (реальная криптография)           │
│  │   └─ Или режим STUB (для тестирования)                    │
│  ├─ Решение о верификации                                     │
│  │   ├─ ЕСЛИ подпись действительна → LED зелёный, переход   │
│  │   └─ ЕСЛИ подпись неверна → LED красный, остановка       │
│  └─ Переход к приложению                                      │
│       └─ jmp 0x08010000 (адрес приложения)                   │
└─────────────────────────────────────────────────────────────────┘

                    ↓ Проверено ↓

┌─────────────────────────────────────────────────────────────────┐
│          ПРИЛОЖЕНИЕ (256+ KB @ 0x08010000)                      │
│                                                                  │
│  ╔══════════════════════════════════════════════════════════╗  │
│  ║           ЯДРО FreeRTOS                                  ║  │
│  ║  Планировщик, задачи, очереди, мьютексы, семафоры      ║  │
│  ╚══════════════════════════════════════════════════════════╝  │
│                                                                  │
│  Задача 1: Подпись FSM       Задача 2: UI/Дисплей            │
│  ├─ Приём UART              ├─ I2C (SSD1306)             │
│  ├─ Разбор команды           ├─ 128x32 пиксела             │
│  ├─ Производная ключа        └─ 4-строчный рендеринг      │
│  ├─ Вычисление SHA-256                                    │
│  ├─ ECDSA подпись            Задача 3: Ввод кнопки        │
│  ├─ Очередь ответа           ├─ Дебаунсинг (30ms)        │
│  └─ Следующее состояние      └─ Подтверждение/Отказ       │
│                                                             │
│  Опцион: LwIP (lwip_zero)    Опцион: USB HID              │
│  ├─ Ethernet                 ├─ 64-байтные отчёты        │
│  ├─ IP стек                  ├─ IN/OUT отчёты             │
│  └─ HTTP сервер              └─ Протокол приложения       │
└─────────────────────────────────────────────────────────────────┘
```

**Ключевой поток данных:**
```
Ввод (UART/HID) → Транспорт подписи → Обработчик команды
    ↓
Управление ключами (В памяти)
    ↓
Хеш SHA-256 → Подпись ECDSA
    ↓
Ответ (UART/HID/Дисплей)
```

**Ключевой поток данных (полная подпись TX):**
```
HTTP POST /tx?amount=1.5&address=1A1z...
    ↓
task_net.c (разбор JSON/форма)
    ↓
tx_request_queue (IPC)
    ↓
task_sign.c (валидация, производная ключа, ожидание кнопки)
    ↓
Пользователь нажимает кнопку
    ↓
Подпись ECDSA (trezor-crypto)
    ↓
Очищение буферов memzero()
    ↓
sign_response_queue (IPC)
    ↓
task_net.c (возврат подписи)
    ↓
HTTP 200 + {signature, tx_id}
```

## 📊 Матрица стека технологий

| Слой | stm32_secure_boot | CryptoWallet |
|---|---|---|
| **CPU** | STM32H743 (Cortex-M7, 480 MHz) | STM32H743 (Cortex-M7, 480 MHz) |
| **OS** | FreeRTOS (8.2.3 или новее) | FreeRTOS (8.2.3+) |
| **Память** | 2 MB Flash / 1 MB SRAM | 2 MB Flash / 1 MB SRAM |
| **Сборка** | GNU Make + arm-none-eabi-gcc | GNU Make + arm-none-eabi-gcc |
| **Загрузчик** | ✅ SHA-256 + ECDSA (CMOX) | ❌ (опционально) |
| **Стек сети** | LwIP (lwip_zero) | LwIP (обязательно) |
| **Библиотека криптографии** | CMOX (ST Micro) | trezor-crypto (Satoshi Labs) |
| **HD кошелёк** | ❌ | ✅ BIP-39/BIP-32 |
| **USB** | ✅ HID (64-байтные отчёты) | ✅ WebUSB (bulk endpoints) |
| **Протокол** | Двойной (UART + HID) | Тройной (UART + WebUSB + HTTP) |
| **Дисплей** | Опциональный SSD1306 | Обязательный SSD1306 |
| **Тестирование RNG** | ❌ | ✅ Комплект Dieharder (НОВОЕ) |
| **Документация** | Польский/Русский | Английский/Русский/Польский |

## 🔗 Сравнение протоколов коммуникации

### stm32_secure_boot - Транспорт UART

```
Формат команды хоста:
┌─────────────────────────────────┐
│ Заголовок кадра: "CMD:"         │
│ Команда: PING, PONG, STATUS,... │
│ Параметры: [опционально]        │
│ Завершитель: \r\n              │
└─────────────────────────────────┘

Пример:

```
