# RNG TEST QUICK START - ACTION CHECKLIST

# This is your step-by-step checklist to fix the RNG capture issue

═══════════════════════════════════════════════════════════════════════════════

ISSUE SUMMARY
─────────────

Error: "device reports readiness to read but returned no data"
Cause: Firmware not configured with USE_RNG_DUMP=1
Fix:   Rebuild firmware with RNG dump enabled

═══════════════════════════════════════════════════════════════════════════════

STEP-BY-STEP CHECKLIST
─────────────────────

□ STEP 1: Understand the Issue
  
  Read: ENABLE_RNG_DUMP.md
  
  This guide explains:
  - What's wrong
  - Why it happened
  - Exactly how to fix it
  - With line numbers and context


□ STEP 2: Edit Makefile (5 minutes)

  cd /data/projects/CryptoWallet
  
  Edit: Makefile
  
  Add this after line 59 (after USE_TEST_SEED):
  
  ---
  # USE_RNG_DUMP=1: output raw RNG data on UART (for Dieharder testing)
  USE_RNG_DUMP ?= 0
  ---
  
  And add this after the USE_TEST_SEED block, before USE_CRYPTO_SIGN:
  
  ---
  ifeq ($(USE_RNG_DUMP),1)
  CFLAGS += -DUSE_RNG_DUMP
  endif
  ---


□ STEP 3: Rebuild Firmware (2 minutes)

  cd /data/projects/CryptoWallet
  make clean
  make USE_RNG_DUMP=1 -j$(nproc)
  
  Expected output:
  - Compiling multiple .c files
  - Linking...
  - Creating binary...
  - Should complete with no errors


□ STEP 4: Flash to Device (1 minute)

  make flash
  
  Or if no flash target:
  
  st-flash write build/firmware.elf 0x08000000
  
  Or if using DFU:
  
  make dfu


□ STEP 5: Verify Firmware (1 minute)

  # Test raw UART output (Ctrl+C to stop)
  python3 -m serial.tools.miniterm /dev/ttyACM0 115200 --raw
  
  Expected: Binary garbage (raw RNG data), NOT text output


□ STEP 6: Run Test (15 minutes)

  source .venv-test/bin/activate
  python3 scripts/test_rng_signing_comprehensive.py --mode rng
  
  Expected:
  - ✓ Checking prerequisites...
  - ✓ pyserial
  - ✓ dieharder
  - ✓ serial port
  
  - ✓ RNG Data Capture from UART
  - Captured: 10 MiB (7.8%) ████░░░░░░...
  - Captured: 128 MiB (100%) ████████████████████...
  
  - ✓ Successfully captured 128 MiB RNG data to rng.bin
  - ✓ Saved analysis to rng_analysis.txt


□ STEP 7: Verify Output Files

  ls -lh rng.bin rng_analysis.txt
  
  Expected:
  - rng.bin ~128 MB (should be exactly 134,217,728 bytes)
  - rng_analysis.txt ~1-5 KB (contains stats)


TROUBLESHOOTING
───────────────

If rebuild fails:
  → See "Common Issues" in ENABLE_RNG_DUMP.md
  → Check ARM toolchain: arm-none-eabi-gcc --version

If flash fails:
  → Disconnect USB, wait 2 sec, reconnect
  → Or see "Flash Instructions" in ENABLE_RNG_DUMP.md

If test still fails:
  → Read: TROUBLESHOOT_RNG_CAPTURE.md
  → Run diagnostic checks from that guide


EXPECTED TIMINGS
────────────────

Makefile edit:      5 minutes (careful reading + edits)
Rebuild:           1-2 minutes (depends on system)
Flash:             1 minute
Verification:      1 minute
Test run:         10-15 minutes
Total:            20-30 minutes


IMPORTANT NOTES
───────────────

• Device must stay connected during entire process
• Rebuild will compile with RNG dump enabled
• Normal UART output will be replaced with binary RNG data
• To restore normal firmware, rebuild without USE_RNG_DUMP=1
• Backup current firmware before rebuilding: cp build/firmware.elf build/firmware.elf.backup
