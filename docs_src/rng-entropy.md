# ГСЧ: источники энтропии (аналог, тепло, TRNG)

Описание текущей реализации и вариантов усиления энтропии.

## Текущая реализация

### 1. STM32 TRNG (аппаратный)

- **Периферия**: RNG (True Random Number Generator)
- **Источник**: аналоговая схема на тепловом шуме в кремнии
- **API**: `HAL_RNG_GenerateRandomNumber()` → 32 бита
- **Инициализация**: `MX_RNG_Init()` + `HAL_RNG_MODULE_ENABLED`

TRNG даёт криптографически стойкие числа, если инициализирован.

### 2. Entropy pool (программный)

```c
s_rng_entropy_pool ^= rng_val;  // XOR с выходом TRNG
s_rng_entropy_pool = LCG(s_rng_entropy_pool);  // 1664525*n + 1013904223
```

- **random_reseed(value)** — добавляет значение в пул (XOR)
- **crypto_rng_init()** — начальное заполнение пула

### 3. Текущий «аналог» при инициализации

Сейчас используется **HAL_GetTick()** (время с момента старта):

```c
uint32_t adc_noise = HAL_GetTick();  // НЕ ADC! Только таймер
// xorshift для рассеивания
adc_noise ^= (adc_noise << 13);
adc_noise ^= (adc_noise >> 17);
adc_noise ^= (adc_noise << 5);
random_reseed(adc_noise);
```

Это не аналоговая энтропия, а псевдослучайность от времени загрузки.

### 4. Fallback при отсутствии TRNG

Если `hrng.Instance == NULL`:

- Используется только LCG от `s_rng_entropy_pool`
- Детерминировано и предсказуемо
- Не подходит для production

## Варианты аналоговой энтропии

### ADC — шум на свободном входе

Незадействованный ADC-канал (floating) даёт шум:

| Источник        | Описание                          |
|-----------------|-----------------------------------|
| Свободный пин   | Термошум, наводки                |
| Внутр. датчик T | Медленно меняется, мало энтропии  |
| Vrefint         | Референсное напряжение + шум      |

```c
// Пример: 8 выборок ADC, младшие биты — энтропия
uint32_t adc_entropy = 0;
for (int i = 0; i < 8; i++) {
    HAL_ADC_Start(&hadc);
    HAL_ADC_PollForConversion(&hadc, 10);
    uint32_t raw = HAL_ADC_GetValue(&hadc);
    adc_entropy ^= (raw << (i * 4)) | (raw >> (32 - i * 4));
}
random_reseed(adc_entropy);
```

### Тепловой шум (косвенно)

- TRNG STM32 уже использует тепловой шум
- ADC на свободном пине — дополнительный источник шума
- Внутренний температурный сенсор — слабый источник, меняется медленно

### Другие источники

| Источник      | Энтропия | Сложность |
|---------------|----------|-----------|
| RNG_DR (TRNG) | Высокая  | Уже есть  |
| ADC floating  | Средняя  | Нужен ADC |
| Jitter таймера| Низкая   | Сложно    |
| Uninitialized RAM | Рискованно | Не рекомендуется |

---

## Рекомендации

1. **Обязательно**: инициализировать TRNG (`MX_RNG_Init`) — основной источник энтропии.
2. **Опционально**: добавить ADC на свободный канал и вызывать `random_reseed(adc_noise)` при старте и периодически.
3. **Не использовать** fallback (LCG без TRNG) в production.

## Схема потоков энтропии

```
                    crypto_rng_init()
                           │
    ┌──────────────────────┼──────────────────────┐
    │                      │                      │
    ▼                      ▼                      ▼
HAL_GetTick()         TRNG (если есть)       ADC (если добавить)
    │                      │                      │
    └──────────► random_reseed() ◄─────────────────┘
                           │
                           ▼
                  s_rng_entropy_pool
                           │
                           ▼
              random_buffer() → XOR с TRNG
```
