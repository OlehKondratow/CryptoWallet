# Сообщения Bitcoin / Ethereum и др.

Описание обработки запросов на подпись для разных криптовалют.

## Формат входящего сообщения

HTTP POST `/tx` (JSON или form-urlencoded):

```json
{
  "recipient": "1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa",
  "amount": "0.001",
  "currency": "BTC"
}
```

| Поле      | Описание                          |
|-----------|-----------------------------------|
| recipient | Адрес получателя                 |
| amount    | Сумма (положительное число)       |
| currency  | Валюта: BTC, ETH, LTC, BCH и др.  |

## Поддерживаемые валюты (whitelist)

Валидация принимает:

- **BTC** — Bitcoin
- **ETH** — Ethereum
- **LTC** — Litecoin
- **BCH** — Bitcoin Cash
- **DOGE** — Dogecoin
- **DASH** — Dash
- **XMR** — Monero

## Валидация адреса получателя

**Текущая реализация** проверяет только **формат Bitcoin**:

| Формат   | Префикс | Длина   | Пример                    |
|----------|---------|---------|---------------------------|
| P2PKH    | `1`     | 26–35   | 1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa |
| P2SH     | `3`     | 26–35   | 3J98t1WpEZ73CNmYviecrnyiWrnqRhWNLy |
| Bech32   | `bc1`   | 42–62   | bc1qxy2kgdygjrsqtzq2n0yrf2493p83kk44h80e |

**Ethereum** (0x + 40 hex) и другие форматы **не проходят** валидацию.

## Что подписывается

Подписывается строка:

```
recipient|amount|currency
```

Пример: `1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa|0.001|BTC`

Далее:
1. SHA-256 от этой строки
2. ECDSA secp256k1 (64 байта, compact r||s)

Это **не** подпись сырой транзакции Bitcoin/Ethereum, а подпись упрощённого сообщения.

## Криптография

| Параметр      | Значение                          |
|---------------|-----------------------------------|
| Кривая        | secp256k1 (Bitcoin, Ethereum)    |
| BIP-32 путь   | m/44'/0'/0'/0/0 (Bitcoin)         |
| Формат подписи| 64 байта (r \|\| s, compact)     |

## Ограничения по валютам

| Валюта | Валидация адреса | BIP-32 путь | Подпись |
|--------|------------------|-------------|---------|
| **BTC** | ✅ P2PKH, P2SH, bc1 | m/44'/0'/0'/0/0 | ✅ secp256k1 |
| **ETH** | ❌ (ожидается 0x...) | ❌ всегда BTC путь | ⚠️ та же кривая, но путь неверный |
| **LTC** | ❌ (L, M, ltc1...) | ❌ | ⚠️ |
| **BCH** | ❌ (bitcoincash:) | ❌ | ⚠️ |
| **XMR** | ❌ (другая схема) | ❌ | ❌ Ed25519, не secp256k1 |

**Итог:** фактически поддерживается только **Bitcoin** (адрес + путь + подпись). Остальные валюты проходят по whitelist, но адрес и путь не соответствуют их спецификации.

## Рекомендации для расширения

1. **Ethereum**: добавить проверку `0x` + 40 hex, путь m/44'/60'/0'/0/0.
2. **Litecoin**: проверка L, M, ltc1; путь m/44'/2'/0'/0/0.
3. **Разные пути**: выбирать BIP-32 путь по полю `currency`.
