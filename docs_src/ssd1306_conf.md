# `Drivers/ssd1306/ssd1306_conf.h`

<brief>Заголовок `ssd1306_conf` задаёт build-time конфигурацию для SSD1306 драйвера: привязывает I2C1 (`hi2c1`) и адрес 0x3C, устанавливает геометрию дисплея 128×32 и включает нужный шрифт (Font 6×8). Эти макросы потребляются исходниками драйвера и вызывающим UI-кодом.</brief>

## Краткий обзор
<brief>Заголовок `ssd1306_conf` задаёт build-time конфигурацию для SSD1306 драйвера: привязывает I2C1 (`hi2c1`) и адрес 0x3C, устанавливает геометрию дисплея 128×32 и включает нужный шрифт (Font 6×8). Эти макросы потребляются исходниками драйвера и вызывающим UI-кодом.</brief>

## Abstract (Synthèse логики)
SSD1306 драйвер на bare-metal обычно компилируется “под конкретный макет” дисплея и под конкретную шину/адрес. `ssd1306_conf.h` — это точка фиксации таких параметров, чтобы `task_display` и `task_display_minimal` могли работать с единым интерфейсом `ssd1306_*`, не занимаясь компиляционными настройками.

## Logic Flow
Это не runtime-модуль, а набор макросов:
| Макрос | Значение | Роль |
|---|---|---|
| `SSD1306_USE_I2C` |  | выбран I2C-бэкэнд |
| `SSD1306_I2C_PORT` | `hi2c1` | привязка к HAL I2C instance |
| `SSD1306_I2C_ADDR` | `(0x3C<<1)` | 7-bit address с учётом shift под I2C |
| `SSD1306_WIDTH`/`HEIGHT` | `128/32` | размеры для буфера/рендеринга |
| `SSD1306_INCLUDE_FONT_6x8` | включено | наличие нужного шрифта |

## Прерывания/регистры
Нет ISR/регистров: только конфигурационные макросы для включения правильного кода драйвера.

## Dependencies
Прямые:
- `hi2c1` из `hw_init.c` (экспортируется в `hw_init.h`).

Куда применяется:
- `task_display.c`, `task_display_minimal.c` (через включение `ssd1306.h` и font).

## Связи
- `hw_init.md` (I2C init + `hi2c1`)
- `task_display.md` (отрисовка поверх SSD1306)
