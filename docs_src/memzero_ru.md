# `memzero.c` + `memzero.h`

<brief>Модуль `memzero` безопасно очищает sensitive буферы (приватные ключи, дайджесты, семена) через volatile-записи, предотвращая оптимизацию компилятора и устраняя следы опасных данных из памяти.</brief>

## Обзор

<brief>Модуль `memzero` безопасно очищает sensitive буферы (приватные ключи, дайджесты, семена) через volatile-записи, предотвращая оптимизацию компилятора и устраняя следы опасных данных из памяти.</brief>

## Абстракция (синтез логики)

`memzero` решает одну критичную проблему безопасности: современные компиляторы могут оптимизировать "бесполезный" `memset(buf, 0, len)`, если компилятор заметит, что `buf` больше не используется после этого. При работе с приватными ключами это опасно — sensitive данные остаются в памяти и могут быть извлечены при атаке. Решение: использовать `volatile` указатель, который компилятор не может оптимизировать. Бизнес-роль: гарантировать, что на каждом пути выхода (успех или ошибка) приватные данные немедленно перезаписаны нулями перед возвратом функции.

## Поток логики

Функция `memzero(void *pnt, size_t len)`:

1. Преобразует указатель в `volatile unsigned char *`
2. Цикл: построчная запись нулей через volatile указатель
3. Возврат

Нет конечного автомата, нет условной логики — просто надёжная перезапись.

## Прерывания/регистры

Нет ISR или регистров: операция записи в памяти на уровне C.

## Времена и условия ветвления

| Параметр | Поведение |
|---|---|
| NULL указатель | no-op (len-- сразу выходит из цикла) |
| len = 0 | no-op (нет итераций) |
| Любой размер буфера | один байт за итерацию, через volatile запись |

## Зависимости

Нет зависимостей: только `<stddef.h>` для `size_t`.

Используется из:
- `task_sign.c` — очищает seed/дайджест/подпись после ошибок или успешного подписания
- `task_security.c` — то же в legacy FSM
- `crypto_wallet.c` — очищает HDNode и приватный ключ
- `wallet_seed.c` — очищает буфер семена
