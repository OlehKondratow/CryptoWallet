# Обзор архитектуры CryptoWallet

## Краткое описание проекта

**CryptoWallet** — это модульная прошивка для подписания Bitcoin-транзакций на микроконтроллере **STM32H743ZI2** (Nucleo-H743ZI2) с использованием **FreeRTOS**. Она интегрирует криптографические операции (BIP-39, BIP-32, ECDSA secp256k1), аппаратную безопасность (MPU, TRNG), сетевые функции (LwIP, Ethernet, WebUSB) и пользовательские интерфейсы (OLED дисплей, кнопки).

## Основные компоненты

### 1. Инициализация (`main.c` / `hw_init.c`)

**Ответственность:**
- Конфигурация MPU (Memory Protection Unit) **перед** HAL_Init (необходимо для безопасности DMA LwIP)
- Включение/отключение кэша в зависимости от регионов памяти
- Инициализация дерева часов (PLL, распределение clock)
- Настройка NVIC для приоритетов прерываний
- Инициализация TRNG для криптографической энтропии
- Гейтирование clock'ов периферии и конфигурация GPIO
- Запуск планировщика FreeRTOS

**Ключевые флаги:**
- `USE_LWIP` — Включить Ethernet + стек LwIP
- `USE_WEBUSB` — Включить режим USB WebUSB
- `USE_CRYPTO_SIGN` — Включить полный ECDSA/BIP-39/BIP-32 (vs. только SHA-256)
- `USE_TEST_SEED` — Использовать hardcoded тестовое семя (только разработка)
- `BOOT_TEST` — Диагностический режим без FreeRTOS (uart debug loop)

### 2. Межзадачное взаимодействие (IPC)

**Глобальные объекты синхронизации:**

```c
// Очереди
QueueHandle_t g_tx_queue;          // task_net → task_sign (wallet_tx_t)
QueueHandle_t g_display_queue;     // task_net → task_display (Transaction_Data_t)

// Группы событий
EventGroupHandle_t g_user_event_group; // task_io → task_sign (подтверждение пользователя)

// Мьютексы
SemaphoreHandle_t g_i2c_mutex;         // Арбитраж шины I2C1
SemaphoreHandle_t g_display_ctx_mutex; // Защита контекста дисплея
```

**Поток данных:**
1. Сетевой пакет (HTTP POST `/tx` или WebUSB endpoint)
2. Парсинг в `task_net` → отправка в `g_tx_queue`
3. `task_sign` получает, отображает транзакцию на OLED
4. Пользователь нажимает кнопку → устанавливает событие в `g_user_event_group`
5. `task_sign` выводит ключ, подписывает транзакцию, отправляет ответ

### 3. Задача дисплея (`task_display.c`)

**Конечный автомат (FSM):**

| Состояние | Режим отображения | Цель |
|---|---|---|
| `DISPLAY_WALLET` | Крипто-тикер + сумма | Показать детали транзакции |
| `DISPLAY_SECURITY` | Статус блокировки + подпись | Подтверждение аутентификации/подписи |
| `DISPLAY_NETWORK` | IP/MAC + статус USB | Информация о сетевом подключении |
| `DISPLAY_LOG` | Прокручиваемый системный лог | Отладка/диагностика |

**Аппаратура:** SSD1306 OLED (128×32, I2C1, addr 0x3C, PB8/PB9)

### 4. Криптографическая интеграция (`crypto_wallet.c` / `task_sign.c`)

**Конвейер подписи:**

```
Ввод пользователя (HTTP POST /tx или WebUSB)
    ↓
Валидация (формат адреса, сумма, whitelist валют)
    ↓
build_hash_input() → "recipient|amount|currency"
    ↓
crypto_hash_sha256() → 32-байтный дайджест
    ↓
Подтверждение пользователя (нажатие кнопки на OLED)
    ↓
get_wallet_seed() → 64-байтное BIP-39 семя
    ↓
crypto_derive_btc_m44_0_0_0_0() → приватный ключ (32 байта)
    ↓
crypto_sign_btc_hash() → ECDSA подпись (64 байта: r||s)
    ↓
memzero() → Очистить конфиденциальные буферы
    ↓
Ответ (USB WebUSB или HTTP JSON)
```

**Криптографические функции (через trezor-crypto):**
- **BIP-39:** Генерация мнемоники из 128-битной энтропии
- **BIP-32:** HD вывод ключей по пути `m/44'/0'/0'/0/0`
- **ECDSA:** secp256k1 подпись с детерминированным RFC6979 k
- **SHA-256:** Хеширование дайджеста транзакции
- **Энтропия:** STM32 TRNG + пул программного LCG

**Условная компиляция:**
- `USE_CRYPTO_SIGN=1` → Полный ECDSA, ~100-150 КБ бинара
- `USE_CRYPTO_SIGN=0` → Только SHA-256, ~5 КБ бинара
- `USE_TEST_SEED=1` → Воспроизводимое тестовое семя (без безопасности)

### 5. Сетевая функциональность (`task_net.c` / `app_ethernet_cw.c`)

**Интеграция LwIP:**
- Стек IPv4 TCP/IP с DHCP клиентом
- Резервный статический IP: `192.168.0.10`
- Синхронизация SNTP (обновление через `time_service_set_epoch()`)
- HTTP сервер для endpoint'а POST `/tx`
- Опциональный WebUSB через `usb_device.c`

**Ethernet PHY:** Интегрирован в STM32H743, DMA буферы в D2 SRAM (кэш должен быть отключен)

**Конечный автомат:**
1. Обнаружение ссылки (PHY прерывание)
2. Запрос DHCP
3. IP получен
4. Запрос SNTP времени
5. Готов к транзакциям

### 6. I/O и пользовательский интерфейс (`task_io.c`)

**Отображение GPIO:**

| Функция | Пин | Режим |
|---|---|---|
| LED зелёный | PB0 | Вывод |
| LED жёлтый | PE1 | Вывод |
| LED красный | PE2 | Вывод |
| Кнопка пользователя | PC13 | Вход (EXTI) |

**Антидребезг:** Аппаратный EXTI с фильтром программного дребезга (20 мс)

### 7. USB и WebUSB (`usb_device.c` / `usb_webusb.c`)

**Конфигурация WebUSB:**
- VID/PID: `0x1209 / 0xC0DE`
- USB скорость: Full-speed (USB 1.1, 12 Мбит/с)
- BOS дескриптор с UUID Platform Capability WebUSB
- Серийный номер динамически генерируется из DEVICE_ID регистров STM32

**Endpoint'ы:**
- EP0: Control (по умолчанию)
- EP1: Data-out (хост → устройство)
- EP2: Data-in (устройство → хост)

### 8. HAL — абстракция микроконтроллера

**Файлы MSP (Microcontroller Support Package):**
- `stm32h7xx_hal_msp.c` — Конфигурация GPIO/clock/NVIC для периферии
- `stm32h7xx_it.c` — Обработчики прерываний (SysTick, Ethernet IRQ)
- `stm32h7xx_it_usb.c` — Обработчик OTG_HS IRQ для WebUSB
- `stm32h7xx_it_systick.c` — Альтернативный SysTick для минимальных LwIP сборок

## Архитектура безопасности

### Memory Protection Unit (MPU)

- **Регион 0:** Flash (код, read-only)
- **Регион 1:** D1 SRAM (общие данные)
- **Регион 2:** D2 SRAM (куча LwIP, кэш отключен для безопасности DMA)
- **Регион 3:** Периферийный I/O (device memory)

**Настройки кэша:**
- D-Cache: Отключен для D2 (регион Ethernet DMA)
- I-Cache: Включен для flash (выборка инструкций)

### Генерация случайных чисел

**Смешивание энтропии:**
1. Аппаратный TRNG STM32H743 (`HAL_RNG_GenerateRandomNumber()`)
2. Пул энтропии программного LCG (параметры Numerical Recipes)
3. XOR комбинация для отбеливания

### Обработка ключевого материала

- BIP-39 семя вычисляется по требованию, никогда не сохраняется
- Приватные ключи выделены в стеке с ограниченным временем жизни
- `memzero()` очищает все конфиденциальные буферы перед возвратом функции
- Нет глобального хранилища ключей в flash прошивки

### Безопасные функции

- Детерминированная ECDSA (RFC6979) предотвращает атаки повторного использования k
- Base58Check валидация для проверки целостности Bitcoin адреса
- Санитизация ввода (regex, whitelist) для запросов транзакций

## Компиляция и система сборки

### Цели Makefile

```bash
make              # Сборка прошивки
make clean        # Очистка артефактов сборки
make docs-doxygen # Генерация API документации (HTML/XML)
make flash        # Программирование STM32 через JTAG/SWD
make monitor      # Консоль отладки UART
```

### Конфигурация сборки

```bash
# Пример: полная функциональная сборка
make USE_LWIP=1 USE_WEBUSB=1 USE_CRYPTO_SIGN=1

# Минимальная сборка (только SHA-256)
make USE_CRYPTO_SIGN=0

# Тестовый режим
make BOOT_TEST=1 USE_TEST_SEED=1
```

## Зависимости

### Аппаратура

- **MCU:** STM32H743ZI2 (ARM Cortex-M7, 480 МГц, 2 МБ Flash, 512 КБ SRAM)
- **Плата:** Nucleo-H743ZI2
- **Дисплей:** SSD1306 OLED 128×32 (I2C1)
- **Сеть:** Интегрированный Ethernet PHY + RJ45

### Программное обеспечение

- **STM32 HAL:** Библиотека аппаратной абстракции
- **FreeRTOS:** API CMSIS-RTOS2
- **LwIP:** Облегченный стек TCP/IP
- **trezor-crypto:** Библиотека Bitcoin криптографии (MIT License)
- **SSD1306 Driver:** Проект stm32-ssd1306

### Внешние проекты

- `/data/projects/STM32CubeH7` — Драйверы HAL
- `/data/projects/stm32-ssd1306` — Библиотека драйвера OLED
- `/data/projects/stm32_secure_boot` — Linker скрипты, конфиг FreeRTOS
- `/data/projects/trezor-firmware` — Эталонная реализация trezor-crypto
