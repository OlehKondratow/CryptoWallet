## 🔧 Флаги Конфигурации Сборки

### Флаги CryptoWallet

```makefile
# Включить/отключить функции при компиляции

USE_LWIP=1              # Включить LwIP + Ethernet (по умолчанию)
USE_CRYPTO_SIGN=1       # Включить подписание ECDSA
USE_TEST_SEED=1         # Использовать жёсткозакодированную мнемонику
USE_WEBUSB=1            # Включить интерфейс WebUSB
USE_RNG_DUMP=1          # Включить статистическое тестирование RNG ⭐ НОВОЕ
SKIP_OLED=1             # Отключить I2C/OLED при зависании шины
LWIP_ALIVE_LOG=1        # Периодический отладочный сигнал о работе

# Примеры:
make                              # Сборка по умолчанию
make USE_CRYPTO_SIGN=1           # С криптографией
make USE_TEST_SEED=1             # С тестовой мнемоникой (автоматически включает крипто)
make USE_RNG_DUMP=1 flash        # Собрать и прошить тестер RNG
make clean all                   # Чистая пересборка
```

## 💡 Варианты Использования

### stm32_secure_boot - Когда Использовать

✅ **Подходит для:**
- Обучение концепциям проверённой загрузки
- Понимание дизайна загрузчика
- Проверка подписей ECDSA
- Коммуникация по двум транспортам (UART + HID)
- Образовательные проекты
- Тестирование криптографических концепций

❌ **Не идеально для:**
- Производственный Bitcoin кошелек (нужен BIP-39/32)
- Сетевая коммуникация
- Сложная логика валидации

### CryptoWallet - Когда Использовать

✅ **Подходит для:**
- Производственное устройство Bitcoin кошелька
- Подписание холодного хранилища
- Мультипротокольная коммуникация (HTTP, WebUSB, UART)
- Валидация качества RNG
- Требуется поддержка BIP-39/BIP-32
- Безопасное управление ключами

❌ **Не идеально для:**
- Обучение дизайну загрузчика
- Простые образовательные проекты
- Минимальные требования к оборудованию

## 🔍 Советы Быстрого Устранения Неполадок

### CryptoWallet

| Проблема | Решение |
|----------|---------|
| Прошивка не удаётся | Попробовать: `make clean && make flash` |
| USB не обнаружен | Проверить: Ошибки в UART, переподключить USB |
| OLED не показывает | Попробовать: `make flash SKIP_OLED=1` |
| HTTP сервер не отвечает | Проверить: Кабель Ethernet, DHCP назначил IP |
| Тесты RNG не удаются | Убедиться: `USE_RNG_DUMP=1` в Makefile |
| Ошибки memzero() | Проверить: Оптимизация компилятора `-O2` |

### stm32_secure_boot

| Проблема | Решение |
|----------|---------|
| Загрузчик не проверяет | Установить: `USE_ECDSA_STUB=1` для тестирования |
| HID не работает | Проверить: USB кабель, дескриптор устройства |
| Вывод UART отсутствует | Проверить: Конфигурация USART3, скорость передачи |
| Инициализация LwIP зависает | Попробовать: Отключить проблемные периферийные устройства |

## 🎯 Что Изменилось: Краткое Резюме

### Основная Тема Обновления: **Инфраструктура Статистического Тестирования RNG**

**Почему?** - Подписание ECDSA зависит от качества RNG:
- Генерация приватного ключа из семени
- Nonce (k) при вычислении подписи
- Если RNG предсказуем → подписи можно подделать
- **Поэтому:** Валидация качества RNG критична

**Что было добавлено:**
1. **Поддержка оборудования**: Режим дампа RNG в прошивке
2. **Комплект тестирования**: Скрипты Python + интеграция Dieharder
3. **Документация**: Полные руководства (3 языка)
4. **Автоматизация**: Скрипты быстрого старта + окружение
5. **Валидация**: Комплексное статистическое тестирование

**Влияние:**
- ✅ Теперь можно валидировать качество RNG
- ✅ Воспроизводимое окружение тестирования
- ✅ Готово к CI/CD
- ✅ Качество производственного уровня
